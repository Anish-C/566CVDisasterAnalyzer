IF ASKED Q&A QUESTIONS
════════════════════════════════════════════════════════════════════════════════

Q: "What's MC Dropout?"
A: "We run the neural network 30 times with random dropout active. Each run 
   gives slightly different predictions. The variation between runs tells us how 
   uncertain the model is."

Q: "How do you generate scenarios?"
A: "We sample 1000 possible damage patterns from the CV predictions, then use 
   K-means clustering to reduce to 50 representative scenarios. Each gets a 
   probability weight."

Q: "Why not just improve CV accuracy?"
A: "Because even 95% accurate still has 5% error. Our approach works with any 
   accuracy if we know the uncertainty. We focus on making robust decisions, 
   not perfect predictions."

Q: "What's the optimization problem?"
A: "Two-stage decision problem. Stage 1: deploy initial resources (before 
   knowing true damage). Stage 2: adapt based on what you learn. Minimize 
   expected casualties."

Q: "Why is this novel?"
A: "Others use CV for predictions OR optimization separately. We formally 
   integrate CV prediction uncertainty into the optimization framework. That's 
   the key contribution."

Q: "How would you deploy this?"
A: "Get satellite images post-disaster → run CV model → generate scenarios → 
   solve optimization → deploy resources. Update as new images arrive."
